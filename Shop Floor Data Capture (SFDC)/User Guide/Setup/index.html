    Setup

    SFDC Setup

    Once installed it’s important to ensure that SFDC is setup to suit business processes and this is largely done within the SFDC Setup screen. This screen can be found through the Search functionality or through the Departments Menu (Departments -> Shop Floor Data Capture -> Setup -> SFDC Setup).

    General FastTab

    The General FastTab needs to be filled in with the Portal Framework Application ID which we covered the creation of in the previous section.

    Data Selection FastTab

    Includes the Data Selection fields, these should be filled with data types that have been setup within the Portal Framework Data Types setup. These Data Types are selected from within the SFDC Screens, with the setup I have above operators would firstly need to select their machine center if there wasn’t a pre-selected default, then their operator code and finally which Production Order they were working on. it is important to note that each of the values within these fields needs to be unique. Also, each must be filled sequentially, meaning that Data Selection 2 cannot be filled in unless Data Selection 1 is and so on.

    Also, here is the Show Ops with no Qty Start which is used to determine whether only operations with a quantity to start are shown or whether they are shown regardless of whether they have a quantity to start or not.

    Next is the Show Prod. Order from Work Center which if ticked will show lines that are assigned to both the current machine center and its parent work center. The decision to tick this will depend on business process, as if there are production planning processes that determine which production orders are completed on which machines in advance then it may not be required for SFDC to display the production order lines for the parent work center.

    Finally, for this FastTab is Exclude Ops from Other Devices, this will filter out operations selected on other devices from the selection screen.

    Output Screen FastTab

    The Allow Scrap Tick box determines whether to display the Scrap button. The Allow By-Products tick box is used for components that can then be re-used, such as metal that can be melted down and used again.

    Allow Device Switching – This should be ticked if the Device is not going to only be on one machine as this allows for the device to be switched to another machine or work center by the Operator. We will discuss this in more detail later in Switching Device.

    Ticking Post Output During Setup Time allows for output to be posted during setup time.

    The Lot Track Output, when ticked, identifies if there is an item tracking line for the production order line, if there is this will be displayed on the output screen. The Lot Track Consumption tick box is used to determine the lot number for a component, this is done through the selection of Lot Info. All consumption made for this component is then taken from that lot number until it is changed within NAV.

    Component Lot Handling – This field should have the options of (blank), Populate when Single Lot or Split by Lot

    A non-blank value can only be selected when "Lot Track Consumption" is ticked

    • Blank Option- will work as per the existing functionality i.e. showing a Lot No. column for the user to populate with the appropriate lot manually.

    • Populate when Single Lot Option- indicates that the system should automatically populate the Lot No. column for a component line when there is a single item tracking line already existing against the prod. order component and therefore we are expecting to consume all stock from a single lot no.

    • Split by Lot Option - If there are already some item tracking lines against a prod. order component it should be possible to split the component lines shown on the portal by each combination of component line no. and lot no. In this case populate the Lot No. column and the Quantity column with the lot numbers and the (sum) quantity that has been assigned to that lot in the item tracking lines.

    This will give the user a clear way to indicate which lot numbers they are consuming when that information is required by the component item.

    This behaviour should be optional by a new setting on the SFDC Setup page. Some customers may want their users to confirm the lot number manually even if there are item tracking lines already assigned to the prod. order component.

    Operators FastTab

    Selecting Show No. of Expected Operators allows for a different number of expected operators in the setup and runtime elements of the operation.

    Ticking the Post Operator Dimensions allows for operator dimensions to be posted with the device capacity entries.

    SFDC Operators have a cost defined on the SFDC Operators screen which is covered later. The Post Operator Time/Cost calculates the time that the operator works on a production order with the Operator Cost and store this on the SFDC Device Log Table.

    Ticking Allow users to Add Notes means that notes can be added within SFDC, not just within NAV.

    Stop Codes FastTab

    The creation of Stop Codes is covered in more detail later, here Stop Codes need to be assigned to each of the stages specified. Each of the codes specified here need to be planned stops due to the nature of the stages.

    Device Status Colour FastTab

    The Output screen can be configured to have a different background colour depending upon the status of the device. It is within the Device Status Colours that these numbers can be specified. These colours help those walking around the shop floor to easily identify the status of the machine and attracts attention to those that are on an unplanned stop.

    Once setup is complete the final step is to run the Calculate Quantity Available to Start, this initialises the Quantity Available to Start on the Production Order Routing Line Table. The SFDC screens can be accessed from this point

    SFDC Operators

    Operators are those working at a given work/machine center on a given production order and will need to be setup to use SFDC functionality. This setup can be found through the Search functionality or through the Departments Menu (Departments -> Shop Floor Data Capture -> Setup)

    Code - Each Operator must have a unique code, this is used for posting the work they have completed.

    Name – Each operator is given a name, this is the field that will appear on the SFDC portal when selecting an operator.

    Machine Center No. – This field is the machine center in which the capacity ledger entries will be posted to if populated.

    Unit of measure – this is the posting unit of measure against Capacity Ledger Entries, in this example we are using minutes.

    Unit Cost – The operators cost per unit of measure again for posting Capacity Ledger Entries this allows for the calculating of the cost of this operator working on the machine

    By highlighting a line within Operators and selecting SFDC Operator Centers, these workers can be made exclusively available to a Work Center Group, Work Center and Machine Center.

    Operator Dimensions

    A dimension for the Operators should also be added to aid in the reporting of SFDC.

    The first step is to add an Operator Dimension to the Dimensions List. This can be accessed using the search functionality or through the Departments Menu (Departments -> Financial Management -> Setup -> Dimensions).

    The Dimension needs to include a Code, Name, Code Caption and Filter Caption as below.

    Once these fields have been completed go to the SFDC Setup Screen and select the Operator Dimensions Action from the Ribbon.

    From here the SFDC Operator Dimensions will open, up to 4 Operator Numbers can be added alongside the Dimension Code.

    Machine Center Setup

    As part of the SFDC install an additional FastTab is added to the Machine Centers screen.

    The first field to select is for Prod. Order Presentation, selecting prompt will allow the operator to choose from a list of available production orders for this machine center. On the other hand, selecting automatic will give the operator the next Production Order on the list, taking away the choice.

    The next field to select is the Process Type, selecting All Lines here will be suitable for machine centers that are used for multiple production orders that are worked on simultaneously. Selecting Single Line will mean that each production order line will need to be selected in isolation with the machine stopped and started again each time.

    If Multiple Lines has been selected within Process Type, the Consolidate Components field can be ticked. Ticking this field will make the Components screen on SFDC appear slightly differently by retrieving the components for all production order lines being worked on simultaneously but will only show each unique item/variant code once.

    Next is the Split-Post Consolidate Components, this gives the option to split and post entries that are being stored on the SFDC Consumptions Entry Table. The Quantity to Post on a Consolidate Components screen should be split proportionally between the production order lines.

    The Show in SFDC tick box needs to be selected for this machine center to show within SFDC.

    Confirm Operator on Output, if ticked a Subform will appear with SFDC with tiles of the operators currently assigned to the prod. order routing line. They will then be prompted to select the operator who has worked on the output.

    The Show Prod. Order from Work Ctr. will override the selection made within the SFDC Setup screen and will show lines that are assigned to both the current machine center and its parent work center.

    Finally, the Use Production Groups may be that certain machine centres do not work on either a single prod. order routing line or a group of prod. order routing lines that all belong to the same prod. order. These scenarios are already supported.

    Instead, it may be the case that a machine works on a group of lines that is grouped by something other than the prod. order no. Depending on the manufacturing process it may be sensible to put lines from different prod. orders into the machine / conveyor belt together.

    For specified Machine Centers, the user should be able to select the production group to work on rather than prod. order.

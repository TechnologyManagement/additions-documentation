    Output Screen

    Logging In

    When SFDC is first loaded depending upon the way in which the Application Device has been setup the Operator may need to select themselves from the log in screen or enter an Access code.

    After selecting their Name or entering an access code and Next the operator will then be directed to the Production Order Line.

    From here they need to select the Production Order they wish to work on and select Next again. The Operator will then be taken to the Output Screen which will essentially act as the Home screen for the Production Order.

    Machine Information

    The Machine Center related to the Machine Center that has been selected by the Operator upon logging into SFDC or the Machine Center that has been set as the default within the Portal Framework Application Devices screen.

    Next the Ready Status shown is the state in which the machine was left in by its last operator. This is also reflected by the colour of the background, if for example the machine was stopped the status would be stopped and the background would be red.

    Finally, the Time Type, interaction with the machine is timed, and the type of time it is set against is shown here which in this example is Setup Time. The amount of time recorded against this Time Type is shown within the SFDC Device Log screen which we will cover later.

    Production Order Information

    The first screenshot above is a typical example of the Production Information provided on the Output screen. The screenshots below show where this same information can be found within NAV.

    Output Information

    Quantity to Start – The Quantity that the Operator has yet to start, this may be a different figure to the Quantity Remaining as items may have been scrapped or there may not be enough components to complete the order

    Quantity Completed – This is how many of the finished Items have been output by the operator.

    Quantity Remaining – The quantity of items that has yet to be completed to finish the order.

    Output Quantity – If a number is entered here and Output is selected this number will be Output by the Operator.

    Production Routing Lines

    If applicable, Production Routing Lines will pull through to the Output Screen, this will show the following.

    Select – This tick box is used in conjunction with the Components, By-Products, Output and Scrap buttons on the Navigation Pane. There can be multiple Production Routing Lines here if they have been setup to run simultaneously.

    Production Routing Line Number – The Production Routing Number is its identifier and is a combination of the Production Order, Item Number and the Routing Line Number.

    Item Number – The Item that is being worked on in the production order.

    Quantity to Start – The Quantity that the Operator has yet to start, this may be a different figure to the Quantity Remaining as items may have been scrapped or there may not be enough components to complete the order

    Quantity Completed – This is how many of the finished Items have been output by the operator.

    Quantity Remaining – The quantity of items that has yet to be completed to finish the order.

    Lot No. Required and Lot No. – If Lot Numbers have been setup for the Production Order these will show here.

    Output Quantity – If a number is entered here and Output is selected this number will be Output by the Operator.

    Operator Information

    The Operator Name and No. are input into SFDC Operators as previously covered. The time displayed is the number of minutes the operator has been working on the machine center. This will continue to count until the machine is stopped by the operator, once stopped the time will be logged alongside its reason code within the SFDC Device Log, which we will cover later.

    Shipment Management Setup

    The Shipment Management Setup page is where general Shipment Management options are defined.

    Scheduling FastTab

    Default Region Code: Used as the default country/region code for Sales Orders on the Shipment Management screen. This is used if the field is empty on the Sales Order.

    Default Delivery Area Code: Used as the default Delivery Area Code when putting the Sales Orders onto the Shipment Management screen. This can be set up as Unassigned for customers that aren’t currently assigned to a specific Delivery Area Code

    Scheduling Time Period: Period for which the Sales Orders will be listed on the Shipment Management screen, i.e. how far ahead do you want to plan. Shipment Management Routes will be created for Orders with a shipment date within the Scheduling Time Period.

    Drop FastTab

    These settings are related to delivering sales orders.

    Show Open orders on Shipment Management – Determines whether open sales/return orders should also be shown on the Shipment Management page. ‘Open’ is a status on the order header.

    Populate Ship Bin on New Route – The Shipment Bin on a Shipment Management Route is the bin where picked goods for shipment will be placed. You may want this field to be automatically populated with a shipment bin, or choose a shipment bin manually before creating picks.

    Collections FastTab

    These settings are related to picking up purchase orders and sales return orders.

    Collection Shipment Method – This field should be set to the Shipment Method Code that will be used on Purchase Orders that will be collected via Shipment Management. The code should indicate the goods are going to be picked up by a Shipment Management Driver. When a Purchase Order is created, and a Shipment Method is entered that matches the Collection Shipment Method in setup, the order will be added to a Shipment Management Route for collection.

    Collection Shipping Agent & Collection Shipping Agent Service Code:

    For Purchase Order Collections: these fields are used to assign Purchase Orders and Sales Return Orders to Delivery Schedules. Purchase Orders will be added to Routes using a Delivery Schedule that matches the Collection Shipping Agent and Collection Shipping Agent Code.

    For Sales Return Orders: Sales Return Orders that are created with a Shipping Agent Code and Shipping Agent Service Code that match the Collections setup will be added to the relevant Shipment Management Route.

    Sorting FastTab

    Allows users to determine the default line sorting order for new picks. There are two options, Deliver Order or Reverse Delivery Order

    On the Delivery Area card, there is a field that determines the Delivery Order.

    The Custom Sorting Order for Picks field can be used when consolidating shipments and creating one pick for the delivery route rather than individual picks for each order.

    When creating a consolidated pick, the pick lines will be sorted in the order specified by the Custom Sorting Order in Shipment Management Setup. This makes it easier for a picker to organise the trolley/pallet they are picking onto.

    Posting FastTab

    The options here determine when invoices are posted for sales orders shipped via the Shipment Management solution. When Warehouse Shipments and Inventory Picks are posted, it is possible to choose whether the sales order should be just shipped or both shipped and invoiced.

    For example, Whse. Shipment Post Setting can be set to Ship or Ship + Invoice. If Ship is chosen, when you use the Post Shipment button on the Shipment Management Routes screen, the shipment is posted, but the sales invoice will not automatically be posted unless Ship + Invoice is selected in setup.

    Report Selections

    Shipment management has its own means of printing documentation relating to the drop off and collection of goods. There is a button on the setup page called Report Selections which when clicked, will open a new window that displays a list of reports that are to be printed when the Print Documentation button is pressed on the Shipment Management page.

    The list page will allow you to enter one or more lines of reports to be printed. You will say which report you want printing out, how you would like the report to be filtered, what order you want that document sorting by (currently only option is delivery order) and then in ascending or descending order.

    NB: Pressing the button will print out all documents, where they print out is run through standard Printer Selections. Meaning each document in this list could be printed to different printers if required.

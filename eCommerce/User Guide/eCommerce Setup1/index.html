    eCommerce Setup

    With the prerequisite complete and the connection tested and established it is time to start setting up your eCommerce data.

    Like some our other additions eCommerce requires a dependency to be installed. Another addition of ours called Additions Config will be installed at the same time.

    NB: You should not Uninstall additions config if you plan on using the eCommerce addition.

    Below are the requirements for setting up the addition and how importing the default configuration will save you time in getting the addition ready for use. The first thing to do is navigate to the eCommerce Setup Page.

    Use the Search Tool to find eCommerce Setup.

    Once on the page you will be presented with several FastTabs, each relating to different aspects of the setup and along the top of the page on the actions ribbon you will come across the Get Default Configuration button

    When the objects are first installed onto the Microsoft Dynamics NAV client, you should run Get Default Configuration action. This will create most of the settings you will see further on in the user guide. By default, it will create the web item number series link, site codes and the inbound and outbound document types.

    General FastTab

    The first field on the General FastTab is the Web Item No Series, this field is populated from the default configuration is run. You can change it, please make sure that if you do that your lines linking back to your No. Series has both a Start and End No, as this field will not populate correctly if you rapidly import your web items.

    Each Web Item created in Microsoft Dynamics NAV needs to have a Web Item No. this field acts as the primary key when linking a web site item to a standard Microsoft Dynamics NAV item.

    NB: The web services uses this bit of information when pulling and pushing data. If the link is broken between the web item for a specific Microsoft Dynamics NAV item, it won’t be shown on the website.

    If you plan on showing stock availability on your website, you can choose at what level the item shows as out of stock. You can set to hide items on the website when they reach a specific level of availability, for example hide items that are Out of Stock.

    To setup your stock availability levels either navigate to the page through the eCommerce Setup page or use the search and find the page Stock Availability Codes.

    Once on the page click on Lines button to see the current setup and make amendments.

    When a customer is created from new registration on the website, this field Customer Template Code, will determine what template to use.

    This field uses standard customer templates, where you can determine default fields such as region code, General Business Posting Group and Customer Discount Groups to reduce manual data entry.

    Images FastTab

    The Image Storage specifies where images uploaded to the website are stored, they can either be stored in the NAV database or Network. When Network is selected you must choose an image network folder.

    Pricing FastTab

    The Price Break Behaviour field determines what quantity of an item can be ordered based on the sales prices for the item. There are two options;

    • Create Quantity 1 Price Break

    • Disable Order Below Min. Price Break

    For example, take a customer that receives a sales discount for an item when they purchase a quantity of 10. If the Price Break Behaviour is Create Quantity 1 Price Break, then the customer can still order the minimum quantity of 1. The default sales prices for the item will be used.

    If the Price Break Behaviour is Disable Order Below Min. Price Break, then the customer will not be allowed to order a quantity under the minimum price break, so they would be required to order a minimum of 10 of the item.

    NB: The price break behaviour can also be set on the web site item. The setup on the web site items takes priority over the eCommerce setup.

    Nonstock Item FastTab

    The final bit of setup relates to Nonstock Items, although these are setup and explained further on in the user guide, you still need to do a small amount of setup in the field below.

    Nonstock items are not item’s you sell they are part of your vendors catalogue.

    It is an item which you can get in or sell separately should a customer enquire about it. When a customer does inquire about it they are presented with a message on screen. It is in the Nonstock Enquiry Response field that you will put in the message you want your customers to see.

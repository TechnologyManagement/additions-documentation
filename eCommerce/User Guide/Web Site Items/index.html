    Web Site Items

    With all the initial setup now complete it is time to create the web data that will be displayed on your website.

    The first step is to navigate to the Web Site Items list page, you can do this by using the search from your role centre

    The Web Site Items list displays all items that are going to be sold online. This list is managed separately to the standard item list, but web site items are directly linked to standard items. Only Web Site Items can be shown on the website. This allows you to have an item list consisting of 1000 items, but if only 200 of these are sold online, the rest do not need to be created as Web Site Items.

    NB: You cannot create a website item that is not already a Microsoft Dynamics NAV item

    On this page you will see all the previous web data you have created as well as buttons along the ribbon to access other areas of web data that needs to be setup.

    This section of the user guide will briefly explain each field on the Web Site Item page, going into further detail about each aspect later on, making sure all the correct web data is created to get the most out of the eCommerce solution.

    To create a new web site item, the corresponding standard item must already be created. From the Web Site Items list, select New from the ribbon to create a new line.

    Enter the Site Code – If you have multiple site codes created for multiple websites the item may belong to one or more of those sites. If this is the case, you would need to enter a new line per site code for the same item.

    Following on from selecting your site code, you need to choose the Item Type. You can use the dropdown to keep it on the default selection of Item or choose the other option of Nonstock Item, further on in the user guide it will detail how to create a Nonstock website item.

    Next enter the Item No. of the item you want to add to the web site. The drop-down option will also you to choose items already created in the standard item list. as mentioned previously you can’t have a web item that is not already setup as a Microsoft Dynamics NAV item number. When you do this, a Web Item No. is assigned.

    You will also notice that as soon as you choose your Item No, the Description will automatically populate (as well as a few other fields) this comes from the Microsoft Dynamics NAV item card and the description on there.

    Other fields on the Web Site Item list are;

    Start/End Date

    For promotional purposes you may want to show a website item or a group of items (maybe belonging to a category) for a short period of time. By setting the start and end date periods you can do this

    No. of Images

    Shows the numbers of images the web item has stored against it. Clicking the field opens the Web Images page, where images can be managed

    Related Info Records

    A web item can have additional information stored against it which will display on the website (under a button labelled View Additional Information). This information can be managed by clicking the Related Info Records field.

    Specification Records

    Multiple description lines can be entered against a website item which will display for the customer viewing the website. This allows you to expand the items description giving the customer more information for them to a decide with.

    Show on Website

    A flag to determine whether the web item should show on the website or not.

    NB: You should NOT delete web items instead use the flag to hide them on the website.

    Last Date/Time Modified

    When any of the fields in this screen are modified the last date modified will update, this allows the job schedule in the background to pick up this item change and flush it to the website.

    No. of Associated Items

    When looking at the details of an item on the website, associated items can be displayed below in the You may also like to buy… section. You select these items from the Web Site Item list.

    No. of Attributes

    Attributes are shown as filters on the website, an item can have multiple filters applied against it so when your customers are trying to find an item within a category they can use the filters to narrow the list down.

    Available Stock

    The information is updated based on the figure on the standard item card but also takes consideration of the quantity of the item on outstanding inbound docume

    Unit Price

    This is a non-editable field. The price is pulled from the Item Card that the web items are linked to.

    Price Break Behavior

    Individual Web Site Items can have their own behaviour on the website this will take priority over the global settings placed on the Website Page or eCommerce setup page, or the settings at parent and child category level (explained further on in the user guide.

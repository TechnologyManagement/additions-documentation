    General FastTab

    Once the eCommerce setup is complete you need to start creating your Web Sites setup. The setup in this page determines how certain actions carried out on the website will be handled in Microsoft Dynamics NAV or vice versa.

    To get to the setup page simply use the search to navigate to the Web Sites Page. On the page you will see a Site Code by the name of Demo. This was created when you ran the Get Default Configuration.

    At this point you have a few options before you start creating all the setup of the site. You can keep the site code as Demo, you can change it or even delete it and create a new one manually.

    NB: The flexibility in our solution means you can create as many site codes as you like. Each with its own setup, items etc.…

    After making the above decision simply click on your Web Site, this will open the Web Sites Card

    Here you can manage the setup of your website. You can create unique codes for each web site if you have more than one, you can then begin defining settings specific to each site.

    Site Codes

    On the FastTab the first field you will notice is the Site Code. This is a unique identifier for each website. When the default config is ran during initial setup, the first site code defaults to DEMO. This can be changed to something more specific to your business before any more setup is completed.

    NB: Should you want to keep the site code as Demo, you can do as it is not visible on the actual website itself it is more to do with the data and how we link it to that site code.

    You can also change the description of each site and choose the language (the default configuration you ran earlier will populate these fields).

    Location Filter for Item Availability

    The next field down is the Location Filter for Item Availability. If you have a Microsoft Dynamics NAV location where stock specifically for your website is held, you can add this location to the website setup so that only stock from that location is shown as available online. If you leave the field blank it will show stock for all locations

    Show Stock Levels

    The field Show Stock Levels allows you to choose whether stock availability is seen by customers viewing the website.

    There are 3 options to choose from:

    • Quantity Available – The precise number of items in stock as per the locations setup in the previous field will be displayed on the website.

    • In/Out Stock – In Stock or Out of Stock will be displayed at the website. The In/Out toggle is governed by the Stock Availability Code.

    • None – When this option is selected, there will be no indication of stock level or availability of the web items.

    Stock-Availability Code

    If you choose option In/Out Stock in Show Stock Levels field, you then need to setup this code. To do this, click on the assist edit button and click on New.

    Select the button Lines to begin setting the stock availability levels.

    On this page, you can change the Minimum Quantity (Base) field based on your customer’s requirements, or even move the order by clicking on the up and down buttons.

    There is an option to Hide Items on Website, this field if ticked will hide all items on the website which have this status, so if they are Out of Stock you may want to hide the items until that item goes back into Limited Stock

    Allow Out of Stock Orders

    The final option on the General FastTab is Allow Out of Stock Orders. If ticked will allow the customer to still place an order for an item that is out of stock.

    The order will come through to pending inbound documents (explained further on in the user guide) in Microsoft Dynamics NAV, it is then up to the business to determine if the order can be fulfilled.

    Shipping FastTab

    Following on from the Accounts FastTab the next tab down is Shipping FastTab. This is where you can change settings related to shipment methods, shipping addresses and delivery promising.

    When placing a web order, customers can choose to collect the goods if this is a service you offer. A Shipping Method needs to be created in Microsoft Dynamics NAV and then populated into the field Shipment Method for Collection. This is then displayed in the Shipping Method field on the Pending Inbound Document and Sales Order in NAV so that the goods are left for the customer to collect.

    The Ship-to No. field is a number series field, this needs to be populated to allow your customers to create one off addresses for their goods to be delivered to. These one-off addresses are outside the list of addresses presented to them during the checkout process, this address come from the customers card in Microsoft Dynamics NAV.

    NB: If the customer needs to use a different shipping address in the future they can change the shipping details during checkout, however the changed address isn’t copied back to the customers Ship-to-Addresses in Microsoft Dynamics NAV, and isn’t saved to be used as part of their checkout process again.

    The Base Calendar code determines the working days that are recognised by NAV. You can create a new calendar and assign non-working days where delivery isn’t available for your customers.

    The next few fields shows how flexible you can be to your customers. The Requested Delivery Date field comes with 2 options:

    • Service Level allows your customers to select fixed price delivery options such as Next Day, 3 Working Days, 7 Working Days etc.

    • Requested Date allows your customers to select a specific date respected by the base calendar working days. The relevant delivery charge is then chosen for the customer. You can see that the service level options are visible but is not editable by your customer. The service level is automatically chosen by the system based on the requested delivery date your customer choses.

    Based on the option you choose above you may also need to setup the delivery window is the date ranges the customer will be able to chose when using the requested delivery date mode. Delivery Window Start determines the earliest available delivery date. Delivery Window End determines the latest date available for the customer. Both of these are set using NAV date formulas.

    Example; an order is placed on 01/05/18.

    • Delivery Window Start = 1D

    • Delivery Window End = 1M

    The available dates for delivery would be between 02/05/18 – 01/06/18 excluding non-working days as defined by the Base Calendar.
